O MVP da plataforma esta pronto em codigo (Next.js 16 + Supabase + PagBank + Bunny + Resend). Supabase migrado (002_pagbank, 003_single_product), webhook PagBank com Basic Auth cadastrado. Faltam Bunny, Resend, conteudo e testes.

Progresso estimado: ~70% · bloqueio principal em Bunny, Resend e conteudo.

Sem essas 4 integracoes, nada roda em producao. Todas exigem acao manual fora do codigo.

Concluido em 20/04/2026

Modelo simplificado: um unico produto avulso (nao e assinatura recorrente) no PagBank. Pagamento unico → acesso vitalicio.

2

Configurar Bunny Stream Bloqueador

Duracao: ~30 min Resp.: Wendrew Pre-req: cartao para cadastro

Hospedagem protegida dos videos das aulas (com tokens e CDN).

  • Criar conta em dash.bunny.net
  • Em Stream, criar Video Library com regiao America do Sul
  • Copiar Library ID, API Key e CDN Hostname
  • Fazer upload de 1 video de teste (5-10 min) para validar player
  • Preencher BUNNY_STREAM_* no .env.local
  • Validar o VideoPlayer localmente acessando uma aula dummy
3

Configurar Resend + verificar dominio Bloqueador

Duracao: ~45 min (+ propagacao DNS) Resp.: Wendrew Pre-req: acesso DNS Hostinger

Disparo de emails transacionais a partir de contato@zelobjj.com.br.

  • Criar conta em resend.com
  • Adicionar dominio zelobjj.com.br em Domains
  • Copiar registros DKIM/SPF/DMARC e aplicar na Hostinger (DNS)
  • Aguardar verificacao (pode levar ate 24h, normalmente < 1h)
  • Criar API Key re_... com escopo de envio
  • Preencher RESEND_API_KEY e RESEND_FROM_EMAIL no .env.local
  • Enviar email de teste via Resend CLI ou Dashboard
4

Reconectar Vercel + GitHub e replicar env vars Bloqueador

Duracao: ~20 min Resp.: Wendrew Pre-req: Steps 1-3 concluidos

Auto-deploy por push e paridade de env vars.

  • Em Vercel → Settings → Git, reconectar repo WendrewDouglas/ZeloBJJ branch master
  • Em Settings → Environment Variables, copiar todas as 15 variaveis do .env.local para Production e Preview
  • Ajustar NEXT_PUBLIC_APP_URL para https://zelobjj.com.br em Production
  • Fazer um commit vazio e push para triggerar deploy
  • Confirmar build verde na Vercel e site funcionando em producao
Atencao: a URL NEXT_PUBLIC_APP_URL precisa bater com a URL real. Em Production, cadastrar PAGBANK_WEBHOOK_USERNAME, PAGBANK_WEBHOOK_PASSWORD e PAGBANK_API_TOKEN com os mesmos valores do painel PagBank.

Riscos & Mitigacoes

RiscoImpactoMitigacao
Verificacao DNS do Resend demorar Alto (bloqueia emails) Iniciar Step 3 primeiro para DNS propagar em paralelo com Steps 1, 2 e 4.
Webhook PagBank nao alcancar a Vercel em Production Critico (assinaturas nao sincronizam) Testar com tunel (cloudflared) em dev; conferir audit_logs apos primeira cobranca real no Step 7.
Bug Next.js 16 de build no Windows Baixo (so local) Nao tentar next build localmente; rodar next dev e deixar build para a Vercel (Linux).
Cliente sem conteudo gravado a tempo Medio (atrasa soft launch) Definir no Step 6 um curso piloto minimo (3-5 aulas) e liberar primeiro antes do catalogo completo.
Vazamento de chaves no repo Critico Confirmar .env.local no .gitignore; rotacionar imediatamente se algo vazar; usar Vercel Env para producao.
Ativacao PagBank demora (validacao KYC/PJ) Alto (bloqueia Fase A) Submeter CNPJ e dados bancarios assim que comecar a Fase A. Conta PJ aprovada e pre-requisito para criar os links de assinatura.
Formato do webhook do PagBank ambiguo (header/secret) Medio Codigo ja tolera 3 nomes de header (x-payload-signature, x-pagbank-signature, x-hub-signature-256). Em homologacao, capturar o header real e ajustar.
